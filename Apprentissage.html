Portfolio d'apprentissage – Parcours IAMSI

Bienvenue dans mon portfolio d’apprentissage. Cette page présente mes réflexions critiques sur les compétences que j’ai développées dans le cadre du BUT Informatique – parcours IAMSI. Je les ai regroupées selon les trois grands axes de formation : optimiser, gérer et organiser. Chaque compétence est accompagnée de projets concrets, de preuves et de mon analyse personnelle.

Optimiser des applications informatiques

AC1.1 – Élaborer et implémenter des spécifications

Projet associé : Stage – IHM pour alimentation

  • Analyse des besoins des techniciens Vitesco pour créer une interface simple, fonctionnelle et adaptée aux tests en laboratoire.
  • Implémentation des comportements attendus des commutateurs dans le code Python (PyQt6) relié à une alimentation programmable Keysight.

Traces : Page du projet, extraits de code Python, capture de l'IHM, rapport de stage.

AC1.3 – Adopter de bonnes pratiques de programmation

Projet associé : Code Game Jam

  • Utilisation de GitHub, gestion collaborative du code, structure modulaire, commentaires utiles, noms explicites.
  • Travail en binôme sur un projet de jeu, avec contraintes de temps et bonne organisation de code pour faciliter les ajouts/bugs fixes.

Traces : GitHub du projet

AC2.4 – Rendre compte de son activité professionnelle

Projet associé : Stage chez Vitesco Technologies

  • Rédaction du rapport de stage structuré : contexte, objectifs, analyse critique, bilan technique et humain.
  • Présentation orale accompagnée d’un support visuel clair, avec retour de l’équipe pédagogique et de mon tuteur industriel.

Traces : Rapport de stage PDF, page projet, plan de soutenance.

AC4.4 – Manipuler des données hétérogènes

Projet associé : Stage – Passerelle LIN

  • Lecture, traitement et conversion de trames issues d’un capteur LIN, transmises via Arduino en USB.
  • Affichage dynamique sur écran LCD et logs en console Python pour différentes structures de données (trames binaires, hexadécimales, texte formaté).

Traces : Captures LCD, extraits de logs, code Arduino, page projet.

Gérer des services informatiques

AC3.3 – Sécuriser les services et données d’un système

Projet associé : Projet Capture The Flag

  • Installation d’un serveur CTF sécurisé via Docker avec gestion des ports, des droits, et des volumes persistants.
  • Rédaction de challenges mettant en avant des failles spécifiques et test de leur exploitabilité par des utilisateurs ciblés.

Traces : Dockerfile, configurations réseau, captures de challenges, page projet CTF.

AC4.1 – Identifier les processus dans une organisation

Projet associé : Serveur Minecraft – Naladia

  • Analyse et structuration du fonctionnement d’un serveur communautaire : rôles du staff, roadmap de contenu, interactions entre développeurs/modérateurs.
  • Utilisation d’un cahier des charges, de Trello et Discord pour suivre les tâches et améliorer la coordination.

Traces : Cahier des charges, page projet Minecraft, schéma de l’organisation.

Organiser des systèmes d’information

Cette compétence transversale s’est développée à travers l’ensemble de mes projets, en particulier :

  • Stage Vitesco : coordination avec les ingénieurs pour recueillir les besoins utilisateurs, planification par étapes, livrables fonctionnels.
  • Naladia : vision d’ensemble d’un écosystème technique (serveur, site web, communauté), gestion des priorités et du développement par versions.
  • Game Jam : organisation rapide et collaborative pour livrer un jeu complet en temps limité.

Traces : Diagrammes de projet, documents de suivi, GitHub, retours d’équipe.

Conclusion

À travers ces projets, j’ai pu développer mes compétences à la fois techniques, humaines et organisationnelles. Le parcours IAMSI m’a permis d’appréhender l’informatique dans sa globalité : développement, sécurité, communication, structuration. Mon stage, en particulier, a été un vrai tremplin pour comprendre l’importance des interactions entre matériel, logiciel et utilisateur dans un contexte professionnel réel.
